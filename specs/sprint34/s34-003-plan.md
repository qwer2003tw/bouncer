#### Trust Module

**File**: `src/trust.py`

修改 `should_trust_approve()`:

```python
from src.config import IP_BINDING_MODE

def should_trust_approve(session_id: str, current_ip: str) -> bool:
    """Check if a trust session allows auto-approval."""
    session = get_trust_session(session_id)

    # ... other checks (expiry, scope, etc.) ...

    # IP binding check
    stored_ip = session.get('ip_address')

    if IP_BINDING_MODE == 'disabled':
        # Skip IP check entirely
        pass
    elif stored_ip and stored_ip != current_ip:
        # IP mismatch detected
        logger.warning(f"IP mismatch: session {session_id}, stored {stored_ip}, current {current_ip}")

        if IP_BINDING_MODE == 'strict':
            # Strict mode: block
            emit_metric('trust_session_ip_blocked', 1)
            logger.error(f"Trust session {session_id} blocked due to IP mismatch (strict mode)")
            return False
        elif IP_BINDING_MODE == 'warn':
            # Warn mode: log + metric, but allow
            emit_metric('trust_session_ip_mismatch', 1)
            # Continue to return True

    return True  # all checks passed
```

### Mode Behavior Matrix

| Mode | Config Value | IP Match | IP Mismatch | Log | Metric |
|------|--------------|----------|-------------|-----|--------|
| Strict | `strict` | ✅ True | ❌ False | error | `trust_session_ip_blocked` |
| Warn (default) | `warn` | ✅ True | ⚠️ True | warning | `trust_session_ip_mismatch` |
| Disabled | `disabled` | ✅ True | ✅ True | - | - |

## IP Binding Modes

Trust sessions bind to the IP address used when creating the session.
You can configure how Bouncer handles IP mismatches:

### Modes

- **`strict`**: IP mismatch blocks the operation (highest security)
  - Use case: High-security production, single IP source
  - Warning: Telegram callbacks and MCP calls have different IPs

- **`warn`** (default): IP mismatch logs warning but allows operation
  - Use case: Mixed Telegram/MCP usage
  - Provides visibility without breaking functionality

- **`disabled`**: No IP check
  - Use case: Development/testing
  - Not recommended for production

### Configuration

Set via environment variable:

```bash
BOUNCER_IP_BINDING_MODE=strict
```

Or in Lambda environment variables (AWS Console).
```
