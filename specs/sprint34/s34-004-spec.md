# Sprint 34-004: Expandable blockquote for long command output

> GitHub Issue: #63
> Priority: P2
> TCS: 11
> Generated: 2026-03-12

---

## Problem Statement

Command execution 產生的長輸出在 Telegram 中不易閱讀。Telegram MarkdownV2 支援 **expandable blockquote** 語法（`**>blockquote expandable**`），可以將長輸出折疊，改善 UX。

### 現狀

`src/telegram.py` 使用 `parse_mode='Markdown'`（MarkdownV1）：

```python
send_telegram_message(
    chat_id=chat_id,
    text=message,
    parse_mode='Markdown'  # MarkdownV1
)
```

**限制**：
1. MarkdownV1 不支援 expandable blockquote
2. 長輸出（>50 行）擠滿 Telegram chat
3. 沒有語法高亮或折疊功能

### 目標

為長 command output 新增 expandable blockquote 支援，使用 Telegram MarkdownV2。

**但範圍限制**：
- **只在 command result / long output 路徑加 expandable blockquote**
- **不全面遷移到 MarkdownV2**（影響範圍太大）

## Root Cause

1. `src/telegram.py` 使用 MarkdownV1（歷史遺留）
2. MarkdownV2 需要更多 escape characters → 全面遷移風險高
3. `src/telegram_entities.py` 已存在（entities mode），但沒有 blockquote builder

## User Stories

**US-1: Collapsed long output**
As a **Bouncer user**,
When a command produces >50 lines of output,
I want to see it in an expandable blockquote,
So that it doesn't clutter my Telegram chat.

**US-2: Short output unchanged**
As a **Bouncer user**,
When a command produces <50 lines of output,
I want to see it in the normal format,
So that I don't have to expand for short results.

**US-3: Manual expand/collapse**
As a **Bouncer user**,
I want to manually expand/collapse long command output in Telegram,
So that I can control what I see.

## Scope

### In Scope

1. 新增 `format_command_output()` helper 到 `src/telegram_entities.py`
2. 判斷 output 長度：>50 行 → expandable blockquote，≤50 行 → normal text
3. 只在 command result notification 路徑使用（不影響其他 notification）
4. 使用 Telegram entities（不用 MarkdownV2 parse_mode）

### Why entities instead of MarkdownV2?

Telegram 支援兩種格式化方式：
1. **parse_mode='MarkdownV2'**: 需要 escape 很多特殊字元（`_*[]()~`#+-=|{}.!`）
2. **entities mode**: 不需要 escape，直接指定 entity type + offset + length

**選擇 entities**：
- 不需要 escape command output（output 可能含任意字元）
- `src/telegram_entities.py` 已存在，可擴展
- 只影響 command result path，不需要全面重構

### Expandable Blockquote Format

Telegram entities 中 expandable blockquote 的 entity type：

```python
{
    'type': 'expandable_blockquote',
    'offset': 0,
    'length': len(text)
}
```

### Out of Scope

- **不全面遷移到 MarkdownV2**（只在 long output 路徑用 entities）
- 不修改其他 notification paths（trust summary, deploy status 等）
- 不新增 syntax highlighting（只是 plain text blockquote）
- 不 configurable threshold（固定 50 行）

## Technical Design

### Entities Builder

**File**: `src/telegram_entities.py`

**New function**:

```python
def format_command_output(output: str, threshold: int = 50) -> tuple[str, list]:
    """Format command output with expandable blockquote if long.

    Args:
        output: Command output text
        threshold: Line count threshold for expandable blockquote (default: 50)

    Returns:
        (text, entities): Formatted text and Telegram entities list
    """
    if not output or output.strip() == '':
        return '(no output)', []

    lines = output.split('\n')
    if len(lines) <= threshold:
        # Short output: no blockquote
        return output, []

    # Long output: expandable blockquote
    entities = [
        {
            'type': 'expandable_blockquote',
            'offset': 0,
            'length': len(output)
        }
    ]
    return output, entities
```

### Integration

**File**: `src/notifications.py` (command result notification path)

**Before**:

```python
message = f"✅ Command completed\n\n{output}\n\nExit code: {exit_code}"
send_telegram_message(chat_id=chat_id, text=message, parse_mode='Markdown')
```

**After**:

```python
from src.telegram_entities import format_command_output

output_text, entities = format_command_output(output)
message = f"✅ Command completed\n\n{output_text}\n\nExit code: {exit_code}"

# Adjust entity offsets (因為有 header text)
header_len = len("✅ Command completed\n\n")
for entity in entities:
    entity['offset'] += header_len

send_telegram_message(chat_id=chat_id, text=message, entities=entities)
```

**Note**: `send_telegram_message` 需支援 `entities` 參數（可能需修改）。
