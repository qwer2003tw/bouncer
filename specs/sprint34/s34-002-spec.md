# Sprint 34-002: Immediate feedback after deploy approval

> GitHub Issue: #117
> Priority: P1
> TCS: 2
> Generated: 2026-03-12

---

## Problem Statement

GitHub issue #117 提出「deploy approval 後缺少 immediate feedback」，但實際檢查後發現：

1. `src/callbacks.py` 的 `handle_deploy_callback()` **已有** immediate feedback（update_message "🚀 部署已啟動"）
2. GitHub issue #117 已被關閉
3. **疑點**：需確認 `handle_execute_callback()` 是否也有類似的 immediate feedback

### 目標

1. 確認 `handle_execute_callback()` 是否缺少 immediate feedback
2. 如果缺少 → 補齊（與 `handle_deploy_callback()` 一致）
3. 如果已有 → 關閉此 task（no-op）

## Root Cause

可能的根因：
1. `handle_execute_callback()` 在實作時沒有參考 `handle_deploy_callback()` 的 immediate feedback 設計
2. 兩個 callback 實作時間點不同，導致不一致
3. GitHub issue #117 可能只描述 deploy 而未提及 execute

## User Stories

**US-1: Execute callback feedback**
As a **Bouncer user**,
When I approve a command execution,
I want to see immediate feedback ("⚙️ 正在執行..." or similar),
So that I know my approval was received before the actual execution completes.

**US-2: Consistent UX across callbacks**
As a **Bouncer user**,
I want deploy and execute approvals to have consistent feedback UX,
So that I have a predictable experience regardless of operation type.

## Scope

### In Scope

1. Read `src/callbacks.py` `handle_deploy_callback()` 確認 immediate feedback 實作
2. Read `src/callbacks.py` `handle_execute_callback()` 檢查是否有 immediate feedback
3. 如果 `handle_execute_callback()` 缺少 → 新增 `update_message()` call
4. 測試驗證：execute approval 後立即看到 feedback message

### Out of Scope

- 不修改 deploy callback（已正確）
- 不改變 feedback message 的內容（保持與現有設計一致）
- 不新增其他 callback types 的 immediate feedback（只處理 execute + deploy）

## Acceptance Criteria

### AC-1: Execute callback has immediate feedback (if missing)
```gherkin
Given a user approves a command execution via Telegram callback
When the callback is processed
Then the approval message is immediately updated to "⚙️ 正在執行..."
And the actual command execution starts after the update
```

### AC-2: Deploy callback still works (no regression)
```gherkin
Given a user approves a deploy via Telegram callback
When the callback is processed
Then the approval message is immediately updated to "🚀 部署已啟動"
And this behavior is unchanged from before
```

### AC-3: Consistent UX
```gherkin
Given both execute and deploy callbacks
When comparing their immediate feedback behavior
Then both update the approval message before starting the operation
And both use a similar message format (icon + status text)
```
