## Problem
`bouncer_deploy` 現有 auto-approve 用 CFN changeset 分析（舊 template），每次都回 "No updates" → 失敗 → 人工審批。

## Solution
在 `bouncer_deploy` 建 approval request 之前，用 git diff 掃描 `template.yaml` 是否有高風險安全變更：
- 無 template 變更 → auto-approve ✅
- 有 template 變更但無高風險 pattern → auto-approve ✅
- 有高風險 pattern → 走人工審批，附上具體的危險項目

## High-risk Patterns（需人工審批）
```python
HIGH_RISK_PATTERNS = [
    (r'Principal\s*:\s*["\']?\*["\']?', "Principal:* — 允許任何人"),
    (r'AuthType\s*:\s*NONE', "Lambda URL AuthType NONE — 無驗證"),
    (r'BlockPublicAcls\s*:\s*false', "S3 BlockPublicAcls disabled"),
    (r'BlockPublicPolicy\s*:\s*false', "S3 BlockPublicPolicy disabled"),
    (r'IgnorePublicAcls\s*:\s*false', "S3 IgnorePublicAcls disabled"),
    (r'RestrictPublicBuckets\s*:\s*false', "S3 RestrictPublicBuckets disabled"),
    # 外部帳號信任（非 trusted accounts）
    (r'arn:aws:iam::(?!(' + '|'.join(TRUSTED_ACCOUNT_IDS) + r')\b)\d{12}:root', "外部帳號信任"),
]
```

### 修改 src/deployer.py

```python
# 新邏輯：git diff 分析
auto_approve_enabled = project.get('auto_approve_deploy', False)
if auto_approve_enabled:
    from template_diff_analyzer import analyze_template_diff
    result = analyze_template_diff(repo_path)
    if result.is_safe:
        # Auto-approve: 直接 start_deploy
        ...
        return mcp_result(req_id, {'status': 'started', 'auto_approved': True, ...})
```


或更簡單：只取兩個版本的 template.yaml 內容做比較（不需要完整 diff）：

## Acceptance Criteria
- [ ] code-only commit → bouncer_deploy 自動 approve，不發 Telegram 審批請求
- [ ] template.yaml 有改但無高風險 → 自動 approve
- [ ] template.yaml 加了 `Principal: "*"` → 走人工審批，Telegram 顯示具體危險項目
- [ ] GitHub API 失敗 → fallback 到人工審批（fail-safe）
- [ ] `auto_approve_deploy: false` → 不分析，直接走人工審批
