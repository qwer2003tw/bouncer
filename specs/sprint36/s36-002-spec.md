## Problem
用戶上傳到 S3 staging bucket 的檔案，目前沒有任何安全掃描。惡意檔案（含 secrets、可執行檔、script）可能被批准後進入正式 bucket。

## Solution
在 `bouncer_upload` / `bouncer_upload_batch` 審批前，掃描上傳的檔案：
1. **檔案類型白名單**：只允許 yaml/json/txt/csv/png/jpg/pdf 等，拒絕 .exe/.sh/.py 等
2. **Secret pattern 掃描**：掃描文字檔是否含 API key、密碼、token 等 pattern
3. **大小限制強化**：超過 10MB 的非圖片/PDF 檔案 → 警告

## Acceptance Criteria
- [ ] .exe/.sh/.bat 檔案 → 自動拒絕，不進審批流程
- [ ] 文字檔含 `AWS_SECRET_ACCESS_KEY=...` → 在審批通知加高風險警告
- [ ] 正常 .yaml/.json/.png → 通過掃描，正常審批流程
- [ ] 掃描失敗（exception）→ fail-safe，允許繼續但在通知加「掃描失敗」警告
- [ ] 不影響現有 trust session 自動上傳流程

## High-risk File Types（自動拒絕）
```python
BLOCKED_EXTENSIONS = {'.exe', '.sh', '.bat', '.cmd', '.ps1', '.vbs', '.dll', '.so', '.dylib'}
```

## Secret Patterns（警告，不自動拒絕）
```python
SECRET_PATTERNS = [
    (r'(?i)aws_secret_access_key\s*=\s*[A-Za-z0-9+/]{40}', "AWS Secret Key"),
    (r'(?i)aws_access_key_id\s*=\s*AKIA[A-Z0-9]{16}', "AWS Access Key ID"),
    (r'(?i)(password|passwd|secret|token)\s*=\s*["\'][^"\']{8,}["\']', "Hardcoded credential"),
    (r'ghp_[A-Za-z0-9]{36}', "GitHub PAT"),
    (r'(?i)private.key', "Private key file"),
]
```
