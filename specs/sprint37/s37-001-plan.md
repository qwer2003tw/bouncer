# s37-001 Plan — fix: changeset count=0 不觸發人工確認

**Issue:** #124

## 改動檔案

### 1. `deployer/notifier/app.py`

**函數：** `handle_infra_approval_request(event)`

**改動說明：**  
從 `event` 讀取 `analysis_error`（via `analyze_result`），根據 `analysis_error` + `change_count` 組合決定 Telegram 通知文字。

```python
# 現有（line 378）
change_count = event.get('change_count', 0)

# 新增（從 SFN analyze_result 取得）
# SFN passes: analyze_result.$: $.analyze_result → entire analyze_result dict is in event
analysis_error = event.get('analysis_error', None)

# 通知文字邏輯
if analysis_error and change_count == 0:
    warning_line = f"⚠️ *Changeset 分析失敗*\n📋 *原因：* `{analysis_error[:200]}`\n無法確認變更範圍，請人工審查。"
elif change_count == 0 and not analysis_error:
    warning_line = "⚠️ *分析結果異常：0 個變更卻未 auto-approve，請人工確認。*"
else:
    warning_line = f"🔧 *變更數量：* {change_count}\n\n⚡ 偵測到 infra 變更，需要人工確認才能繼續部署。"
```

**注意：** SFN payload 將整個 `analyze_result` 傳入 Lambda：
```yaml
# deployer/template.yaml line 725
Payload:
  analyze_result.$: $.analyze_result
```
`event` 即為此 Payload，所以 `analysis_error` 在 `event.get('analysis_error', None)` 可讀到。
