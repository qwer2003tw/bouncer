## 改動項目

### 1. DDB 更新（透過 Bouncer 執行）

**Pre-condition：先確認 `ztp-files` 現有 config**

```bash
aws dynamodb get-item \
  --table-name bouncer-projects \
  --key '{"project_id": {"S": "ztp-files"}}' \
  --region us-east-1
```

驗證以下欄位存在且正確：
- `git_repo` → `https://github.com/qwer2003tw/ztp-files`（或類似 GitHub URL）
- `stack_name` → `ztp-files-dev`（或實際 stack name）
- `default_branch` → `master`

**更新指令：**

```bash
aws dynamodb update-item \
  --table-name bouncer-projects \
  --key '{"project_id": {"S": "ztp-files"}}' \
  --update-expression "SET auto_approve_deploy = :v" \
  --expression-attribute-values '{":v": {"BOOL": true}}' \
  --region us-east-1
```

**若 `git_repo` 未設定，同時更新：**

```bash
aws dynamodb update-item \
  --table-name bouncer-projects \
  --key '{"project_id": {"S": "ztp-files"}}' \
  --update-expression "SET auto_approve_deploy = :aad, git_repo = :gr" \
  --expression-attribute-values '{
    ":aad": {"BOOL": true},
    ":gr": {"S": "https://github.com/qwer2003tw/ztp-files"}
  }' \
  --region us-east-1
```

### 2. 驗證（透過 Bouncer 執行）

## 注意事項

1. **DDB `auto_approve_deploy` 必須是 `BOOL` 型別**，不能是字串 `"true"`  
   Python `project.get('auto_approve_deploy', False)` 對非空字串 `"true"` 也是 truthy，  
   但 DDB 型別不一致會造成未來的 update_project_config 問題。

2. **`template_diff_analyzer` 只看最新 commit 的 diff**  
   如果 ztp-files 最近有多次 commit 累積的 template 變更，第一次 auto-approve 部署可能 **fail-safe** 走人工審批（GitHub API 取 HEAD vs parent，只看 1 個 commit）  
   → 正常行為，第一次部署後之後的 code-only deploy 就能 auto-approve。

3. **`github_pat_secret` 使用預設值** `'sam-deployer/github-pat'`  
   ztp-files 的 `git_repo_owner` = `qwer2003tw`，應有相同的 GitHub PAT 存取權。
