# s37-004 Spec — fix: auto_approved 通知補帳號欄位

**Issue:** #122
**Sprint:** 37
**Type:** Bug Fix

---

## 問題描述

auto_approved（safelist）通知不包含帳號資訊，Steven 看到通知時不知道命令是在哪個 AWS 帳號執行的。  
特別是 cross-account 操作（dev account 992382394211 / 1st account 841882238387 / default 190825685292）時，  
「自動執行」通知沒有帳號欄位，降低了可審計性。

---

## 根因分析

**File:** `src/mcp_execute.py`，`_check_auto_approve()` 函數

### 現有通知文字（約 line 629-640）

```python
_notif_text = (
    f"\u26a1 *自動執行*\n\n"
    f"\U0001f916 *來源：* {escape_markdown(ctx.source or '(unknown)')}\n"
    f"{reason_line}"
    f"\U0001f4cb *命令：*\n```\n{ctx.command[:300]}\n```\n\n"
    f"\u2705 *結果：*\n```\n{result_preview}\n```"
)
```

**缺少：** `ctx.account_id` 和 `ctx.account_name`

### 可用資料

`ctx` 是 `ExecuteContext` dataclass（line 183-184），包含：
- `ctx.account_id: str` — AWS 帳號 ID（e.g. `992382394211`）
- `ctx.account_name: str` — 帳號名稱（e.g. `Dev`, `Default`）

這兩個欄位在 `_check_auto_approve()` scope 內完全可用。  
`log_decision()` 在同一函數中已正確傳入（line 648-649）。

### 對比其他通知（已有帳號欄位）

**Grant auto-approved 通知（line ~556-570）：**
```python
response_data = {
    'account': ctx.account_id,
    'account_name': ctx.account_name,
    ...
}
```

**人工審批通知（`notifications.py`，`send_approval_request`）：**
```python
mb.text("🏦 ").bold("帳號：").text(" ").code(account_id).text(f" ({account_name})").newline()
```

auto_approved 通知是系統中唯一缺少帳號欄位的通知。

---

## 修法方向

在 `_notif_text` 加入帳號行：

```python
account_line = (
    f"🏦 *帳號：* `{ctx.account_id}` ({escape_markdown(ctx.account_name or '')})\n"
    if ctx.account_id else ""
)
_notif_text = (
    f"⚡ *自動執行*\n\n"
    f"🤖 *來源：* {escape_markdown(ctx.source or '(unknown)')}\n"
    f"{account_line}"
    f"{reason_line}"
    f"📋 *命令：*\n```\n{ctx.command[:300]}\n```\n\n"
    f"✅ *結果：*\n```\n{result_preview}\n```"
)
```

## Acceptance Scenarios

### Scenario 1: 指定帳號執行 → 通知顯示帳號
**Given** 呼叫 `bouncer_execute` 帶 `account="992382394211"`  
**When** 命令在 safelist，自動執行  
**Then** Telegram 通知包含：  
```
🏦 帳號: 992382394211 (Dev)
```

### Scenario 2: 預設帳號執行 → 通知顯示預設帳號
**Given** 呼叫 `bouncer_execute` 不帶 `account` 參數  
**When** 命令在 safelist，自動執行  
**Then** Telegram 通知包含：  
```
🏦 帳號: 190825685292 (Default)
```
（帳號 ID 從 `DEFAULT_ACCOUNT_ID` 環境變數取得）

### Scenario 3: 通知被 throttle → 不發通知（帳號欄位不影響 throttle 邏輯）
**Given** `_should_throttle_notification('auto_approve')` 回傳 `True`  
**When** 命令在 safelist，自動執行  
**Then** 通知被跳過（現有行為不變）

### Scenario 4: `account_id` 為空字串 → 不顯示帳號行
**Given** 某些邊緣情況 `ctx.account_id = ''`  
**When** 命令自動執行  
**Then** 帳號行不顯示（空字串 guard：`if ctx.account_id else ""`）

---

## Edge Cases

| Edge Case | 預期行為 |
|-----------|---------|
| `account_name` 含特殊 Markdown 字元（如 `Dev-2nd`） | `escape_markdown()` 正確 escape |
| `account_id` 為 `None`（理論上不應發生） | `if ctx.account_id` guard 保護，不顯示帳號行 |
| `account_name` 為空字串 | 顯示 `帳號: 190825685292 ()` 或省略括號（可選） |
| throttle 後首次顯示 → 帳號欄位正確 | throttle 不影響欄位邏輯 |
