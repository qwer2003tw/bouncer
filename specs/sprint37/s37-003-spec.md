# s37-003 Spec — feat: ZTP Files auto-approve

**Issue:** #125
**Sprint:** 37
**Type:** Feature

---

## 問題描述

`ztp-files` 專案每次部署都需要人工審批，即使是 code-only 的 Lambda function 更新也一樣。  
`bouncer` project 已有 `auto_approve_deploy=True`，`ztp-files` 應享有同樣的 auto-approve 流程：
- Code-only 變更（Lambda code + SAM lifecycle）→ **自動批准**，直接 deploy
- 有 infra 變更（IAM、API Gateway、S3 等）→ 仍走人工審批

---

## 根因分析（現狀）

### Auto-approve 機制

**File:** `src/deployer.py`，`mcp_tool_deploy()` line 967-1000：

```python
auto_approve_enabled = project.get('auto_approve_deploy', False)

if auto_approve_enabled:
    from template_diff_analyzer import analyze_template_diff
    git_repo = project.get('git_repo', '')
    github_pat_secret = project.get('github_pat_secret', 'sam-deployer/github-pat')
    
    diff_result = analyze_template_diff(git_repo, branch, github_pat_secret)
    
    if diff_result.is_safe:
        # Auto-approve: start deploy directly
        ...
    else:
        # High-risk → human approval (with context)
        ...
```

### `template_diff_analyzer.analyze_template_diff()` 邏輯

**File:** `src/template_diff_analyzer.py`

1. 從 `git_repo`（格式 `https://github.com/{owner}/{repo}`）解析 owner/repo
2. GitHub API 取 HEAD commit 與 parent commit
3. 比對 `template.yaml` diff（只看 added lines）
4. 掃描 `HIGH_RISK_PATTERNS`（`Principal:*`、`AuthType:NONE`、S3 public block disabled 等）
5. 回傳 `TemplateDiffResult(is_safe=True/False, ...)`

### ZTP Files 現狀

`bouncer-projects` DDB table 的 `ztp-files` item **缺少** `auto_approve_deploy` 欄位（或為 `False`）。  
→ `project.get('auto_approve_deploy', False)` 回傳 `False`  
→ 每次都走人工審批流程。

### 所需 DDB 更新

```json
{
  "project_id": "ztp-files",
  "auto_approve_deploy": true,
  "git_repo": "https://github.com/qwer2003tw/ztp-files",
  "git_repo_owner": "qwer2003tw"
}
```

（`git_repo` 和 `git_repo_owner` 應已存在，需確認；`auto_approve_deploy=true` 是新增的）

---

## 修法方向

**不需要改 code**，只需要：

1. **確認** `ztp-files` 在 DDB 有正確的 `git_repo`（`https://github.com/...`）  
2. **Bouncer AWS CLI** 更新 DDB item，設定 `auto_approve_deploy=true`

具體 DDB 更新指令（透過 Bouncer 執行）：
```bash
aws dynamodb update-item \
  --table-name bouncer-projects \
  --key '{"project_id": {"S": "ztp-files"}}' \
  --update-expression "SET auto_approve_deploy = :v" \
  --expression-attribute-values '{":v": {"BOOL": true}}' \
  --region us-east-1
```

### 備選方案（若 `git_repo` 未設定）

若 `git_repo` 欄位空或格式錯誤，`template_diff_analyzer` 會回傳 `TemplateDiffResult(is_safe=False, error=...)`，  
`deployer.py` 的 else branch 會加 `context = "[changeset 分析失敗...]"` 並走人工審批，**安全降級**。

## Acceptance Scenarios

### Scenario 1: Code-only 部署 → auto-approve
**Given** `ztp-files` DDB item 有 `auto_approve_deploy=true`  
**And** 最近一次 commit 只修改了 Lambda function code（不改 template.yaml）  
**When** 觸發 `bouncer_deploy ztp-files`  
**Then**  
- `template_diff_analyzer` 分析 template.yaml diff → 無變更 → `is_safe=True`  
- 部署直接 start（`status=started, auto_approved=True`）  
- Telegram 發出 auto-approve 通知（`send_auto_approve_deploy_notification`）  
- **不發** 人工審批請求

### Scenario 2: 修改 template.yaml（增加 IAM Role）→ 仍觸發人工確認
**Given** `ztp-files` 有 `auto_approve_deploy=true`  
**And** 最近一次 commit 在 template.yaml 新增 `AWS::IAM::Role`  
**When** 觸發 `bouncer_deploy ztp-files`  
**Then**  
- `template_diff_analyzer` 偵測到 template.yaml 有 non-Lambda 變更 → `is_safe=False`  
- Context 加上「[需審批：高風險 infra 變更 — ...]」  
- 發出人工審批 Telegram 通知

### Scenario 3: 修改 template.yaml 但只加 Lambda（SAM AutoPublishAlias）→ auto-approve
**Given** `auto_approve_deploy=true`  
**And** template.yaml diff 只有 `AWS::Lambda::Function` Code 屬性變更  
**When** 觸發部署  
**Then** auto-approve（`is_safe=True`）

### Scenario 4: GitHub API 失敗 → fail-safe 人工審批
**Given** `auto_approve_deploy=true`  
**And** GitHub API 回傳 503  
**When** 觸發部署  
**Then**  
- `template_diff_analyzer` 回傳 `TemplateDiffResult(is_safe=False, error="HTTP 503 ...")`  
- Context 加入錯誤說明  
- 人工審批通知發出（fail-safe）

---

## Edge Cases

| Edge Case | 預期行為 |
|-----------|---------|
| `git_repo` 格式錯誤（不是 GitHub URL） | `analyze_template_diff` 回傳 error，fail-safe 走人工審批 |
| `git_repo` 為空字串 | 同上，error path |
| DDB `auto_approve_deploy` 型別不是 bool（字串 `"true"`） | `project.get('auto_approve_deploy', False)` 在 Python 中 non-empty string 為 truthy，`if auto_approve_enabled:` 仍觸發 — **需注意 DDB 寫入時用 BOOL 型別** |
| `template.yaml` 不存在於 repo | `analyze_template_diff` 回傳 `is_safe=True`（無 template 變更）→ auto-approve |
| 部署是 first commit（無 parent） | `_get_latest_commits` 偵測到無 parent，`analyze_template_diff` 回傳 `error='no parent'`，fail-safe |
