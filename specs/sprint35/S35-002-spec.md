# S35-002: Fix Pre-existing Test Failures — TestOrphanApprovalCleanup

## 概述

修復 `tests/test_callbacks_main.py` 中 `TestOrphanApprovalCleanup` class 的兩個失敗測試：
1. `test_execute_telegram_failure_ddb_no_record_returns_error`
2. `test_execute_telegram_exception_ddb_no_record_returns_error`

這些測試的目的是驗證：當 Telegram 通知失敗時，DDB 不應該留下 orphan approval record，且 MCP tool 應回傳 error 狀態。

## 背景

### 根本問題

當前測試 mock 了舊的 `telegram.send_telegram_message()` API，但：
1. **實際使用的 API：** `notifications.py` 使用 `telegram.send_message_with_entities()`（Sprint 24 entities migration）
2. **回傳值格式：** 測試 mock 回傳 `{}`（空 dict），但 `send_approval_request` 預期 `{'ok': True}` 或 `NotificationResult(ok=True, message_id=...)`
3. **Error handling：** 當 Telegram 回傳空 dict 時，`send_approval_request` 在 `notifications.py` 中應該：
   - 回傳 `NotificationResult(ok=False, message_id=None)`
   - 觸發 `mcp_execute.py` 中的 cleanup 邏輯

### 當前測試邏輯（有問題）

**檔案：** `tests/test_callbacks_main.py` (Lines 929-971)

```python
@patch('telegram.send_message_with_entities')
def test_execute_telegram_failure_ddb_no_record_returns_error(self, mock_telegram, mock_dynamodb, app_module):
    """Telegram 失敗（empty response）→ DDB 無 record，回 error ✅"""
    mock_telegram.return_value = {}  # Telegram failure returns empty dict

    # ... trigger bouncer_execute ...

    # 預期：content['status'] == 'error'
    # 實際：可能回傳 'pending_approval'（因為 DDB 已寫入）
```

### 問題所在

1. **Mock 不正確：** `send_message_with_entities` 應該回傳 `{'ok': False}` 而非 `{}`
2. **Cleanup 邏輯未觸發：** `mcp_execute.py` 中的 `_submit_for_approval` 在 Telegram 失敗時應該：
   - 檢查 `notified.ok == False`
   - 刪除已建立的 DDB record
   - 回傳 `mcp_error(...)`

3. **Exception handling 路徑不同：** `test_execute_telegram_exception` 測試 exception，但 exception 可能被 `send_approval_request` 內部 catch，不會傳播到 `_submit_for_approval`

```python
@patch('telegram.send_message_with_entities')
def test_execute_telegram_failure_ddb_no_record_returns_error(self, mock_telegram, mock_dynamodb, app_module):
    """Telegram 失敗（empty response）→ DDB 無 record，回 error ✅"""
    # Fix: send_message_with_entities 回傳 {'ok': False} 表示失敗
    mock_telegram.return_value = {'ok': False}  # 或 {}（視 notifications.py 實作）

    event = {
        'rawPath': '/mcp',
        'headers': {'x-approval-secret': 'test-secret'},
        'body': json.dumps({
            'jsonrpc': '2.0',
            'id': 901,
            'method': 'tools/call',
            'params': {
                'name': 'bouncer_execute',
                'arguments': {
                    'command': 'aws ec2 stop-instances --instance-ids i-p1-1b',
                    'trust_scope': 'test-scope',
                    'reason': 'P1-1 regression failure test',
                }
            }
        }),
        'requestContext': {'http': {'method': 'POST'}}
    }

    result = app_module.lambda_handler(event, None)
    body = json.loads(result['body'])
    content = json.loads(body['result']['content'][0]['text'])

    assert content['status'] == 'error', f"Expected error, got: {content}"
    assert 'Telegram' in content.get('error', '') or 'notification' in content.get('error', '').lower()

    # Verify DDB record was cleaned up (no orphan record)
    scan_result = mock_dynamodb.Table('clawdbot-approval-requests').scan(
        FilterExpression='#cmd = :cmd AND #status = :status',
        ExpressionAttributeNames={'#cmd': 'command', '#status': 'status'},
        ExpressionAttributeValues={
            ':cmd': 'aws ec2 stop-instances --instance-ids i-p1-1b',
            ':status': 'pending_approval',
        }
    )
    orphan_items = [i for i in scan_result.get('Items', [])
                    if i.get('reason') == 'P1-1 regression failure test']
    assert len(orphan_items) == 0, f"Orphan DDB records found: {orphan_items}"
```
