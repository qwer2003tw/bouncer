# S35-003: Approval Timeout Notification

## 概述

在 `bouncer_execute` / `bouncer_deploy` 審批請求即將過期前 N 秒（建議 60 秒），發送 Telegram 提醒通知給 Steven，避免請求默默過期。

## 背景

當前行為：
- `bouncer_execute` / `bouncer_deploy` 請求發出後，TTL = 300 秒（5 分鐘）
- 如果 Steven 未在 5 分鐘內批准，請求過期（DynamoDB TTL 刪除）
- **問題：** 無任何提醒告訴 Steven「有請求要過期了」

使用者痛點：
- Steven 可能正在忙其他事，錯過 5 分鐘視窗
- 請求過期後，AI agent 需要重新發送請求（浪費時間）
- 無法區分「Steven 主動拒絕」和「Steven 錯過視窗」

## 解決方案：EventBridge Scheduler

### 架構

```
bouncer_execute/deploy MCP call
  → create DDB record (TTL = now + 300)
  → send Telegram notification
  → post_notification_setup() (sprint7-002 已有)
    → create EventBridge Scheduler one-time schedule (at TTL - 60s)

EventBridge Scheduler trigger (at TTL - 60s)
  → invoke Lambda with {"action": "expiry_warning", "request_id": "..."}
  → Lambda checks DDB record status
    → if still "pending_approval": send Telegram warning
    → else: skip (already approved/denied)
```

### 為何選擇 EventBridge Scheduler

**已有基礎：** Sprint 7-002 已建立 `scheduler_service.py` 和 `create_expiry_schedule()`，用於 expiry cleanup。只需新增 warning schedule。

**對比其他方案：**

| 方案 | 優點 | 缺點 | 成本 |
|------|------|------|------|
| EventBridge Scheduler | - 精確時間觸發<br>- 已有基礎設施 | - 需要兩個 schedule（warning + cleanup） | 前 14M 次/月免費 |
| DynamoDB TTL stream | - 自動觸發 | - TTL 時間不精確（48 小時內）<br>- 需要 Streams 費用 | $0.02/100 萬讀取 |
| Lambda polling | - 簡單 | - 需要持續 Lambda 運行<br>- 浪費資源 | 高（持續 Lambda 費用） |

**推薦：** EventBridge Scheduler（成本最低，時間最精確）

```python
def create_warning_schedule(
    request_id: str,
    expires_at: int,
    warning_seconds_before: int = 60,
    telegram_message_id: Optional[int] = None,
) -> None:
    """Create a one-time EventBridge Scheduler to send expiry warning.

    Args:
        request_id: The Bouncer request ID
        expires_at: Unix timestamp when the request expires (TTL)
        warning_seconds_before: Seconds before expiry to send warning (default: 60)
        telegram_message_id: Optional Telegram message ID to include in warning

    Raises:
        ClientError: If schedule creation fails
    """
    import os
    import json
    from botocore.exceptions import ClientError
    from aws_lambda_powertools import Logger

    logger = Logger(service="bouncer")
    scheduler_client = _get_scheduler_client()

    warning_time = expires_at - warning_seconds_before

    # Skip if warning time is in the past
    if warning_time <= int(time.time()):
        logger.warning(
            "Warning time is in the past, skipping schedule creation",
            extra={
                "src_module": "scheduler",
                "operation": "create_warning_schedule",
                "request_id": request_id,
                "expires_at": expires_at,
                "warning_time": warning_time,
            }
        )
        return

    schedule_name = f"bouncer-warning-{request_id}"
    lambda_arn = os.environ.get("LAMBDA_ARN", "")

    payload = {
        "action": "expiry_warning",
        "request_id": request_id,
        "expires_at": expires_at,
        "telegram_message_id": telegram_message_id,
    }

    try:
        scheduler_client.create_schedule(
            Name=schedule_name,
            ScheduleExpression=f"at({_unix_to_schedule_time(warning_time)})",
            Target={
                "Arn": lambda_arn,
                "RoleArn": os.environ.get("SCHEDULER_ROLE_ARN", ""),
                "Input": json.dumps(payload),
            },
            FlexibleTimeWindow={"Mode": "OFF"},
            State="ENABLED",
        )
        logger.info(
            "Expiry warning schedule created",
            extra={
                "src_module": "scheduler",
                "operation": "create_warning_schedule",
                "request_id": request_id,
                "schedule_name": schedule_name,
                "warning_time": warning_time,
            }
        )
    except ClientError as exc:
        logger.error(
            "Failed to create expiry warning schedule: %s",
            exc,
            extra={
                "src_module": "scheduler",
                "operation": "create_warning_schedule",
                "request_id": request_id,
                "error": str(exc),
            }
        )
        raise
```

```python
def handle_expiry_warning(event):
    """Handle EventBridge Scheduler expiry warning trigger.

    Args:
        event: {
            "action": "expiry_warning",
            "request_id": "...",
            "expires_at": 1234567890,
            "telegram_message_id": 123  # optional
        }

    Returns:
        {"statusCode": 200} on success
    """
    import time
    from db import table
    from telegram import send_telegram_message_silent

    request_id = event.get('request_id', '')
    expires_at = event.get('expires_at', 0)
    telegram_message_id = event.get('telegram_message_id')

    if not request_id:
        logger.warning("expiry_warning: missing request_id", extra={"src_module": "expiry", "operation": "warning"})
        return {'statusCode': 400, 'body': 'Missing request_id'}

    # Fetch DDB record
    try:
        resp = table.get_item(Key={'request_id': request_id})
        item = resp.get('Item')
    except ClientError as exc:
        logger.error("Failed to fetch DDB record for %s: %s", request_id, exc, extra={"src_module": "expiry", "operation": "warning", "request_id": request_id, "error": str(exc)})
        return {'statusCode': 500, 'body': 'DDB error'}

    # Check if still pending
    if not item:
        logger.info("Request %s not found (already deleted by TTL?)", request_id, extra={"src_module": "expiry", "operation": "warning", "request_id": request_id})
        return {'statusCode': 200, 'body': 'Request not found'}

    status = item.get('status', '')
    if status != 'pending_approval':
        logger.info("Request %s already %s, skipping warning", request_id, status, extra={"src_module": "expiry", "operation": "warning", "request_id": request_id, "status": status})
        return {'statusCode': 200, 'body': 'Already processed'}

    # Send Telegram warning
    remaining_seconds = expires_at - int(time.time())
    if remaining_seconds < 0:
        remaining_seconds = 0

    # Build warning message
    command_preview = item.get('command', item.get('action', ''))[:80]
    source = item.get('source', 'Unknown')

    text = (
        f"⏰ *審批請求即將過期*\n\n"
        f"🆔 `{request_id}`\n"
        f"📋 `{command_preview}`\n"
        f"🤖 來源：{source}\n\n"
        f"⏱ 剩餘時間：{remaining_seconds} 秒\n"
        f"請儘快審批，否則將自動過期。"
    )

    try:
        send_telegram_message_silent(text)
        logger.info("Expiry warning sent for %s", request_id, extra={"src_module": "expiry", "operation": "warning", "request_id": request_id})
    except Exception as exc:
        logger.error("Failed to send expiry warning for %s: %s", request_id, exc, extra={"src_module": "expiry", "operation": "warning", "request_id": request_id, "error": str(exc)})
        return {'statusCode': 500, 'body': 'Telegram error'}

    return {'statusCode': 200, 'body': 'Warning sent'}
```

- **request_id 驗證：** Lambda handler 檢查 request_id 是否存在
